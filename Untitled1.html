CMB lensing map validation¶

Author: Patricia Larsen

In this jupyter notebook I am validating the mass sheets input to a CMB lensing map. These mass sheets are:

  • z<3: the same lens plane mass maps used to create the galaxy lensing maps. These are created with a modified DTFE method from downsampled particle lightcones in cosmo-DC2.
  • z>3: log-normal realisations of mass sheets, provided by Joachim Harnois-Deraps.

For now I combine these in a simple Born approximation, scaled with the lensing kernel and added without ray-tracing. This is provided in separate z<3 and z>3 maps so that we can properly check scaling consistency and swap out the higher redshift mass sheets if needed. Once these maps are properly validated we can apply ray-tracing.

We clearly have an issue here, which looks like a scaling mismatch combined with lack of correlations between high redshift lens planes - we need to do a few things, validate that the low redshift values are correct, check the conventions of the outputs, and work out what is happening with the mass at high redshift. An obvious place to start is to look at the cross-correlation between CMB lensing and the DC2 galaxies
